Questions

      
Things developers actually ask.

    


    

      

        
        

          
An outlet claim is a factual assertion made by the publication itself — in the headline, lede, or body copy written by the journalist or editorial staff. These claims reflect the outlet's own editorial judgment and count toward outlet scoring.

          
An attributed claim is a factual claim quoted or paraphrased from a named source: a politician, expert, spokesperson, or other third party. These claims reflect what the source said, not what the outlet asserts. They do not count toward outlet scoring.

          
This distinction is central to the Verum Signal methodology. Read more →

        

      


      

        
        

          The API returns one of the following verdict labels on each claim, in lowercase: supported, plausible, corroborated, overstated, disputed, not_supported, not_verifiable, and opinion. The first three indicate the claim is well-supported by evidence. overstated, disputed, and not_supported indicate degrees of weakness. not_verifiable and opinion are excluded from outlet scoring — the first means independent verification was not possible, the second means the claim is a value judgment rather than a factual assertion. Full definitions and scoring weights are documented at /methodology.
        

      


      

        
        

          Each claim is evaluated against independent sources under a documented methodology. The methodology is publicly versioned, and every API record is stamped with the version that produced its verdict. Read the methodology →
